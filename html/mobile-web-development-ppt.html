倍率（ios）= 像素密度等级（android）= 设备像素比（DPR）

分辨率增长速度大于屏幕尺寸 -> PPI变大也就是一英寸中的像素变多 -> 像素密度等级上升

  • 1倍（163 ppi、1～3gs）
  • 2倍（326 ppi、4～6s）
  • 3倍（401 ppi、plus系列）

  • ldpi [0.75倍]
  • mdpi [1倍]
  • hdpi [1.5倍]
  • xhdpi [2倍]
  • xxhdpi [3倍]
  • xxxhdpi [4倍]

pt/dp

pt（ios），dp（android）—— 出现了那么多不同尺寸的手机，怎么在各个终端上达到与设计稿同样的效果呢？

在1倍（160ppi ）情况下，1pt等于1px，也就是说160pt = 160px =1英寸，在倍率为2（ 320 ppi）,1pt 等于2px ，160pt ＝ 320px =1英寸。

高清方案 by 阿里巴巴

根据设备屏幕的DPR（设备像素比，又称DPPX，比如dpr=2时，表示1个CSS像素由4个物理像素点组成） 动态设置 html 的font-size, 同时根据设备DPR调整页面的缩放值，进而达到高清效果

dpr=1，则html的font-size为50px，此时 1rem = 50px
dpr=2，html的font-size为100px，此时 1rem = 100px
dpr=3，html的font-size为150px，此时 1rem = 150px

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
